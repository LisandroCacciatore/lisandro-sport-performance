# Script: Evaluación de Preparación del Cliente

Usar antes de enviar una propuesta formal.

**Paso previo:** Identificar el buyer. Los criterios de preparación son distintos
según quién es el interlocutor y quién aprueba la compra.

---

## Checklist para Coordinador de Rendimiento / Preparador Físico

Puntuar: ✅ Confirmado / ⚠️ Parcial / ❌ Ausente

```
1. CALENDARIO DEFINIDO
   ¿El atleta tiene al menos una fecha de competencia en los próximos 90 días?
   ✅ Sí, con fecha específica
   ⚠️ "Hay algo en el próximo trimestre" (vago)
   ❌ No hay calendario o no compite próximamente

2. REGISTRO EXISTENTE
   ¿Ya registra algún dato de carga, aunque sea informalmente?
   ✅ Planilla, app, o registro sistemático
   ⚠️ "A veces anoto algo"
   ❌ No registra nada

3. BRECHA ARTICULABLE
   ¿Puede describir un problema concreto en su proceso de decisión?
   ✅ Describió una instancia de decisión fallida o incierta
   ⚠️ "Creo que podría mejorar" (sin caso específico)
   ❌ No identifica ningún problema

4. AUTORIDAD DE DECISIÓN
   ¿La persona con quien hablo puede tomar la decisión de compra?
   ✅ Coordinador con presupuesto propio o del club
   ⚠️ "Tengo que consultarlo con el director / la academia"
   ❌ PF sin poder de decisión

5. VENTANA DE TIEMPO
   ¿Hay una ventana de 14 días disponible antes del próximo torneo?
   ✅ Sí, hay margen suficiente
   ⚠️ "Hay un torneo en 3 semanas" (alcance reducido posible)
   ❌ Competencia en menos de 2 semanas
```

### Interpretación

| Puntaje | Acción |
|---|---|
| 5 ✅ | Enviar propuesta inmediatamente |
| 4 ✅ + 1 ⚠️ | Enviar propuesta, aclarar el punto parcial |
| Criterio 4 ⚠️ (PF que necesita consultar) | Equipar con materiales para la conversación interna |
| 3 ✅ o menos | Resolver los puntos ausentes primero |
| Criterio 1 o 2 ❌ | Derivar a contenido IronLog. Revisitar en 30–60 días. |

## Checklist para Director Deportivo

```
1. PROBLEMA INSTITUCIONAL ARTICULABLE
   ¿Puede describir una situación donde la falta de criterio compartido
   generó un conflicto, lesión no anticipada, o pregunta que no pudo responder?
   ✅ Sí, con caso concreto
   ⚠️ "Podría mejorar la coordinación del staff" (vago)
   ❌ No identifica ningún problema institucional

2. PRESIÓN EXTERNA REAL
   ¿Hay alguien — comisión directiva, sponsor, padres — pidiendo justificaciones?
   ✅ Sí, hay presión activa
   ⚠️ "Eventualmente nos van a preguntar"
   ❌ No hay presión externa visible

3. STAFF CON AL MENOS 2 PREPARADORES
   ¿El problema aplica a un cuerpo técnico, no a un PF individual?
   ✅ Sí, hay múltiples roles
   ⚠️ "Somos un preparador y un asistente"
   ❌ Un solo preparador → derivar a Auditoría VCV individual

4. AUTORIDAD Y PRESUPUESTO
   ¿Esta persona puede aprobar el gasto de USD 500/mes?
   ✅ Sí, tiene autoridad directa
   ⚠️ "Lo tengo que pasar por la comisión" (probable sí, con tiempo)
   ❌ No tiene autoridad de gasto

5. HORIZONTE DE TRABAJO
   ¿Hay disposición para un proceso de 3 meses?
   ✅ Sí, entiende que el cambio institucional lleva tiempo
   ⚠️ "Depende de cómo vaya el primer mes"
   ❌ Busca solución inmediata → ofrecer Auditoría como mes 0
```

### Interpretación

| Puntaje | Acción |
|---|---|
| 5 ✅ | Enviar propuesta de Sistema de Gobernanza |
| Criterio 3 ❌ (un solo PF) | Redirigir a Auditoría VCV Tier 1 |
| Criterio 5 ⚠️ | Proponer Auditoría como "mes 0" de la Gobernanza |
| Criterio 4 ⚠️ | Preparar documento ejecutivo para facilitar la aprobación interna |

## Checklist para Dueño de Academia

```
1. COMPETENCIA EN EL MERCADO
   ¿Hay competidores usando datos como argumento de venta o retención?
   ✅ Sí, hay presión competitiva activa
   ⚠️ "Creo que van a empezar a hacerlo"
   ❌ No hay competencia visible con datos → urgencia baja

2. INTERLOCUTORES QUE PREGUNTAN POR EL PROCESO
   ¿Padres, atletas adultos, o inversores preguntan por la metodología?
   ✅ Sí, es frecuente
   ⚠️ "A veces alguien pregunta"
   ❌ No hay demanda de justificación

3. EVALUACIONES FÍSICAS EXISTENTES
   ¿La academia hace tests o evaluaciones con periodicidad definida?
   ✅ Sí, con frecuencia definida
   ⚠️ "Hacemos algo, no muy sistemático"
   ❌ No hay evaluaciones → el sistema no tiene base para operar

4. AUTORIDAD Y PRESUPUESTO
   ¿Esta persona puede aprobar la inversión?
   ✅ Es el dueño con autoridad total
   ⚠️ "Tengo un socio" (probable sí, con conversación interna)
   ❌ No tiene autoridad de gasto

5. OBJETIVO COMERCIAL CLARO
   ¿Puede articular cómo el sistema le ayudaría a vender más,
   cobrar más caro, o retener mejor?
   ✅ Conecta el sistema con un resultado de negocio específico
   ⚠️ "Me parece que sería bueno tener datos"
   ❌ No conecta el sistema con ningún resultado concreto
```

### Interpretación

| Puntaje | Acción |
|---|---|
| 5 ✅ | Enviar propuesta con foco en diferenciación comercial |
| Criterio 3 ❌ (sin evaluaciones) | Derivar a simuladores IronLog. Revisitar cuando tengan datos. |
| Criterio 5 ⚠️ | Trabajar el argumento de ROI antes de enviar propuesta |
| Criterios 1 y 2 ❌ | Urgencia inexistente. Contenido de posicionamiento primero. |

## Qué hacer cuando el PF necesita consultar

Esta situación es frecuente y tiene un proceso específico:

```
1. Completar el diagnóstico técnico con el PF.
2. Preguntarle: "¿Qué pregunta va a hacer tu director / el dueño primero?"
3. Preparar el argumento para esa pregunta en el idioma del decisor.
4. Ofrecer un documento de 1 página:
   - El problema identificado (en lenguaje del buyer decisor, no técnico)
   - El entregable concreto (qué van a tener al final)
   - El precio y el tiempo
   - Una frase de ROI en el idioma del decisor

El PF no convence a su director con argumentos técnicos.
Lo convence con el argumento que el Director ya entiende:
orden institucional, credibilidad, reducción de riesgo.
```
