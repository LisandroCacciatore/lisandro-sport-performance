# Script: Diagnóstico de Brecha Operativa

Usar cuando el usuario necesite analizar a un cliente potencial
o cuando llegue con descripción de un problema de staff técnico.

**Paso previo obligatorio:** Identificar el buyer antes de empezar.
El perfil determina qué preguntas hacer y cómo interpretar las respuestas.

---

## Identificación del buyer

```
¿Quién es el interlocutor?
  → Coordinador de Rendimiento / Head of Performance
     → Diagnóstico técnico completo. Las 4 preguntas en orden.

  → Preparador Físico de equipo o categoría
     → Diagnóstico técnico completo.
        Nota: puede no ser quien aprueba la compra.
        Ver "Caso especial: PF sin autoridad de compra" al final.

  → Director Deportivo / Manager del Club
     → Diagnóstico de gobernanza. Usar preguntas de la sección específica.
        El argumento es institucional, no técnico individual.

  → Dueño de Academia Deportiva
     → Diagnóstico de diferenciación. Usar preguntas de la sección específica.
```

---

## Diagnóstico para Coordinador de Rendimiento / Preparador Físico

### Pregunta 1 — La regla de decisión

> "¿Qué dato te llevaría a cambiar la carga de tu atleta esta semana?"

```
¿La respuesta incluye un número, umbral o condición específica?
  → SÍ: Puede haber un sistema. Ir a Pregunta 2.
  → NO: Brecha Tipo A confirmada. Ir a propuesta de Auditoría VCV.
```

### Pregunta 2 — El registro

> "¿Qué datos estás registrando actualmente, y con qué frecuencia?"

```
¿El registro es sistemático (aunque sea una planilla)?
  → SÍ: Hay material para trabajar. Ir a Pregunta 3.
  → NO: Pre-cliente. Necesita instalar el hábito de registro primero.
         Derivar a simuladores IronLog. Revisitar en 2–3 semanas.

¿Menciona más de 4–5 métricas distintas?
  → SÍ: Señal de Brecha Tipo B. Ir a Pregunta 4.
  → NO: Probable Brecha Tipo A. Continuar.
```

### Pregunta 3 — La brecha en consecuencia

> "¿Hubo alguna instancia en el último ciclo donde el atleta llegó
> a una competencia sin estar donde esperabas?"

```
¿Puede describir una instancia concreta?
  → SÍ: Brecha con consecuencia visible. El ROI es articulable.
         Registrar: qué pasó, cuándo, cuántas semanas afectó.
         Esta instancia es la base del argumento de propuesta.

  → NO (nunca pasó): Puede ser que el sistema funcione.
         "Si nunca hubo un problema visible, quizás la prioridad es otra.
         ¿Hay alguna pregunta sobre el sistema de carga que no podés responder?"

  → NO (no recuerda): La brecha puede estar normalizada.
         "¿Podría haber pasado sin que te dieras cuenta?"
```

### Pregunta 4 — La gobernanza (solo si hay señal de Tipo B)

> "¿Todos los entrenadores del staff ajustan la carga con el mismo criterio?
> Si le pregunto a otro entrenador, ¿daría la misma respuesta que vos?"

```
"Probablemente no" / "No lo sé"
  → Brecha Tipo B confirmada.
     Propuesta: Gobernanza (Tier 2) o Auditoría como puerta de entrada.

"Sí, tenemos criterio común"
  → "¿Ese criterio está escrito o es tácito?"
     Si tácito: riesgo ante cambio de staff. Hay brecha latente.
     Si escrito: no hay brecha de gobernanza. Evaluar Brecha Tipo A individual.
```

## Diagnóstico para Director Deportivo

El Director no responde preguntas técnicas. El diagnóstico es institucional.

### Pregunta de apertura

> "Si le preguntás a dos preparadores del staff qué dato los llevaría a cambiar
> la carga esta semana, ¿te darían la misma respuesta?"

```
"Probablemente no" / "No lo sé"
  → Brecha Tipo B confirmada. Propuesta: Gobernanza (Tier 2).

"Sí, todos trabajamos igual"
  → "¿Ese criterio está escrito o es tácito?"
  → Si tácito: riesgo ante cambio de staff. Brecha latente.
```

### Seguimiento para cuantificar el costo institucional

> "¿Hubo lesiones en el último semestre que nadie anticipó?
> ¿Cómo respondió el club cuando alguien preguntó por qué pasó?"

```
Si describe un conflicto o falta de respaldo:
  → Argumento de ROI: "Un criterio institucional escrito convierte
     esa conversación en algo que podés responder con datos, no con intuición."
```

## Diagnóstico para Dueño de Academia

El Dueño piensa en competencia y valor percibido. El diagnóstico es de diferenciación.

### Pregunta de apertura

> "Cuando un padre o un atleta pregunta por qué el programa de entrenamiento
> está diseñado de la forma en que está, ¿podés mostrarle el proceso detrás?
> ¿O es una respuesta verbal que depende de que vos estés presente?"

```
"No tenemos eso documentado" / "Les explicamos"
  → Oportunidad de diferenciación. El proceso existe pero no es vendible.

"Tenemos reportes"
  → ¿Incluyen criterio de decisión o solo datos?
  → Si solo datos: el proceso sigue siendo invisible.
```

### Seguimiento para cuantificar la oportunidad

> "¿Hay academias en tu mercado usando datos como argumento de venta?
> ¿Cómo te estás diferenciando de ellas hoy?"

```
Si menciona competencia con datos:
  → Urgencia real. El sistema es defensa de posición.

Si no hay competencia visible:
  → Oportunidad de ser primero. Argumento de ventaja temprana.
```

## Casos especiales

### PF sin autoridad de compra

El PF encuentra el contenido pero el Director o Dueño firman el presupuesto.

Completar el diagnóstico técnico con el PF, luego:
> "¿Qué pregunta va a hacer tu director / el dueño primero?
> ¿Precio, evidencia de resultado, o cómo encaja con lo que ya tienen?"

Darle el argumento exacto para la conversación interna.
El PF no convence con argumentos técnicos: convence con el argumento
que el Director ya entiende (orden, credibilidad, reducción de riesgo).

### Academia con Director Técnico y PF distintos

Dos interlocutores con necesidades diferentes:
- El **PF** quiere criterio operativo → Auditoría VCV
- El **Director Técnico** quiere gobernanza y reporting → Gobernanza

Si se habla con el Director primero: enmarcar en términos de consistencia del staff y riesgo institucional.
Si se habla con el PF primero: enmarcar en términos de autonomía y criterio propio. Luego equiparlo para la conversación con el Director.

### Cliente que ya hizo una Auditoría y el problema persiste

El problema evolucionó de Tipo A a Tipo B.
La regla existe pero el staff no la aplica consistentemente.
Propuesta: Sistema de Gobernanza.

### Calendario de competencia en menos de 3 semanas

La Auditoría sigue siendo posible (14 días).
Foco limitado a la VCV del bloque de tapering, no del sistema completo.
Mencionar en la propuesta: "alcance reducido al bloque actual."
