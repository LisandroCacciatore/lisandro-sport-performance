# Ejemplo: Documento de Brecha (Fase 1)

Este es un ejemplo completo de un documento de Fase 1 para un preparador físico
de una academia de tenis junior, usando el método Wendler 5/3/1.
Sirve como referencia de formato, densidad, y lenguaje.

---

## Versión técnica (para Coordinador / PF)

```
AUDITORÍA VCV — DOCUMENTO DE BRECHA
Academia [NOMBRE] — Preparador: [NOMBRE]
Fecha: 15/04/2026

─────────────────────────────────────────

BLOQUE ACTUAL
  Método:           Wendler 5/3/1
  Semana del ciclo: Semana 1 de 4 (semana de volumen 5x5+)
  Objetivo:         Preparación para torneo regional — 6 semanas
  Próxima comp.:    29/05/2026 — Torneo Regional Sub-18

─────────────────────────────────────────

LO QUE SE REGISTRA
  - AMRAP de semana 3 de cada ciclo (los últimos 3 ciclos documentados)
  - RPE subjetivo post-sesión (escala 1–10, registrado 3x/semana)
  - Peso corporal matutino (inconsistente — 2 de cada 3 semanas)

LO QUE SE DECIDE
  - El TM del ciclo siguiente se sube según el protocolo estándar de Wendler
    (+2.5kg tren superior / +5kg tren inferior), independientemente del AMRAP
  - No hay umbral escrito para determinar cuándo el TM está sobreestimado
  - La decisión de mantener o reducir TM ocurre solo cuando el atleta falla
    la semana pesada o reporta fatiga alta varias sesiones seguidas

─────────────────────────────────────────

LA BRECHA
  "El AMRAP de semana 3 se registra pero no tiene una regla escrita que defina
   cuándo su resultado debe detener o revertir la progresión del TM."

VARIABLE CRÍTICA IDENTIFICADA
  AMRAP de Semana 3 — Sentadilla (ejercicio principal del ciclo actual)

EVIDENCIA DE CONSECUENCIA
  Ciclo de febrero 2026:
  - AMRAP semana 3: 3 reps (Sentadilla — 87.5kg)
  - TM siguiente ciclo: subió a protocolo estándar (+5kg → 92.5kg)
  - Semana pesada del ciclo siguiente: atleta falló el 3er intento al 95%
  - Consecuencia: se perdió la semana pesada + 1 semana de recalibración
  - El torneo de ese mes quedó con el atleta fuera de pico

─────────────────────────────────────────

PRÓXIMO PASO
  Fase 2 comienza el 17/04/2026.
  Entregable: Esquema IF → THEN con umbrales operativos para el AMRAP de Sentadilla,
  aplicable al ciclo que culmina el 24/04/2026.
```

## Versión ejecutiva (para Director Técnico de la academia)

```
AUDITORÍA VCV — RESUMEN EJECUTIVO DE DIAGNÓSTICO
Academia [NOMBRE]
Fecha: 15/04/2026

─────────────────────────────────────────

CONTEXTO
  El cuerpo técnico trabaja con el atleta en preparación para el Torneo Regional
  Sub-18 del 29 de mayo. El ciclo actual tiene 6 semanas hasta la competencia.

─────────────────────────────────────────

EL PROBLEMA IDENTIFICADO
  El staff registra los resultados de entrenamiento cada ciclo, pero no existe una regla
  escrita que defina cuándo esos resultados requieren ajustar la progresión de carga.

  Las decisiones de ajuste se toman cuando el problema ya es visible
  (el atleta falla un levantamiento o reporta fatiga alta), no antes.

CONSECUENCIA CONCRETA
  En febrero, el atleta llegó al torneo con una carga de entrenamiento mayor
  a la que podía sostener. La señal estaba en los datos 3 semanas antes,
  pero no había una regla que la convirtiera en acción.
  El atleta compitió fuera de su pico de rendimiento.

─────────────────────────────────────────

QUÉ SE VA A RESOLVER EN LAS PRÓXIMAS DOS SEMANAS
  Al terminar la Auditoría, el staff tendrá un protocolo escrito que define:
  - qué resultado de entrenamiento indica que la carga debe revisarse
  - qué acción específica tomar en cada caso
  - quién es responsable de registrar y decidir

  El protocolo lleva menos de 15 minutos por semana y no requiere
  que el consultor esté presente para aplicarlo.

─────────────────────────────────────────

PRÓXIMO PASO
  Fase 2 comienza el 17/04/2026.
  Entregable: Protocolo de decisión semanal aplicable a partir del lunes 21/04/2026.
```

## Notas sobre este ejemplo

**Por qué la brecha está bien escrita:**
Es específica, tiene el nombre de la métrica, y tiene consecuencia verificable en los datos.
No dice "el sistema no funciona" — dice qué parte del sistema está ausente.

**Por qué la versión ejecutiva no tiene números:**
El Director Técnico no toma decisiones de TM. Toma decisiones de proceso y recursos.
La versión ejecutiva conecta el problema con una consecuencia que el Director sí puede ver
(el atleta fuera de pico en un torneo).

**Lo que este ejemplo no hace:**
- No critica al preparador físico
- No propone la solución en el documento de diagnóstico
- No exagera el problema ni usa lenguaje alarmista
