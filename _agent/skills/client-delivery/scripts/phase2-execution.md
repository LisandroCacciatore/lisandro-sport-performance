# Fase 2: Diseño — Ejecución Día a Día (Días 5–10)

Objetivo de la fase: construir el esquema IF → THEN con umbrales operativos
para la VCV identificada en Fase 1. El resultado es una regla que el staff
puede aplicar sin consultar al consultor.

Entregable: Esquema IF → THEN (ver plantilla en `resources/phase-templates.md`).

---

## Día 5 — Llamada de diseño: definir los umbrales

**Duración:** 45–60 minutos
**Requisito previo:** Documento de brecha aprobado por el cliente.

**Agenda:**

```
0–10 min   → Confirmar la VCV y el contexto actual
  - "¿Algo cambió desde que hablamos?"
  - "¿El [VCV] se registró esta semana? ¿Cuál fue el valor?"

10–30 min  → Definir los umbrales en conjunto con el cliente
  Objetivo: el cliente propone, el consultor refina y valida

  Preguntas para elicitar los umbrales:
  1. "¿Cuándo sabés que el atleta está en un buen momento? ¿Qué número o resultado te indica eso?"
  2. "¿Cuándo empezás a preocuparte? ¿Qué número o resultado cambia tu atención?"
  3. "¿Cuándo sabés que hay que actuar sí o sí? ¿Qué número o resultado te fuerza la decisión?"

  Regla: los umbrales deben venir del cliente primero.
  Si el cliente no puede nombrar un número: proponer uno y verificar.
  "¿Si el AMRAP es de 3 reps, eso te preocupa?" → afinar desde la respuesta.

30–45 min  → Definir las acciones THEN
  Para cada zona de umbral:
  - "¿Qué hacés ahora en esa situación?"
  - "¿Eso es lo que debería pasar, o hay algo que nunca llegaste a implementar?"
  
  Las acciones deben ser específicas y ejecutables:
  ✅ "Reducir TM un 5% en el próximo ciclo"
  ❌ "Evaluar si ajustar la carga"

45–60 min  → Primer borrador del esquema en conjunto
  Escribir la regla IF → THEN en voz alta con el cliente.
  El cliente la valida en el momento.
```

**Después de la llamada:**
- Transcribir el esquema a la plantilla formal
- Marcar cualquier punto ambiguo que quedó pendiente

---

## Día 6 — Refinamiento del esquema y casos borde

**Actividad principal:** Trabajar los casos que el Día 5 no resolvió completamente.

**Preguntas para identificar casos borde:**

```
- ¿Qué pasa si el atleta estaba enfermo esa semana? ¿La regla igual aplica?
- ¿Qué pasa si el dato es inconsistente con la semana anterior?
- ¿Hay condiciones externas (viaje, estrés, calor) que deberían cambiar el umbral?
```

**Criterio para incluir casos borde:**
Solo incluir lo que ocurre con frecuencia real. Si es un caso hipotético que nunca pasó, no agregarlo.
La regla simple que funciona el 90% del tiempo es mejor que la regla compleja que intenta cubrir todo.

**Redactar las notas de contexto:**
- Condiciones que modifican la aplicación de la regla
- Diferencias entre ejercicios o atletas si aplica
- Qué hacer si el dato no se registró esa semana

---

## Día 7 — Redacción del protocolo de 15 minutos

**Actividad principal:** Convertir el esquema IF → THEN en un protocolo operativo semanal.

El protocolo de 15 minutos no es el esquema — es la **secuencia de pasos** para aplicarlo.

**Estructura del protocolo:**

```
Paso 1: Registrar [VCV] al finalizar [sesión específica del día X]
  → Dónde: [planilla / app / formato acordado]
  → Quién: [nombre o rol]
  → Tiempo estimado: [X minutos]

Paso 2: Comparar el valor con el umbral
  → Referencia: [dónde está escrito el umbral]
  → Tiempo estimado: [X minutos]

Paso 3: Aplicar la regla IF → THEN correspondiente
  → Referencia: [dónde está el esquema]
  → Si hay duda sobre la zona: [criterio de desempate]

Paso 4: Documentar la decisión
  → Dónde: [planilla / app / formato acordado]
  → Qué registrar: [dato, zona, decisión tomada]
  → Tiempo estimado: [X minutos]

Total estimado: [suma de pasos] minutos
```

**Verificación:**
Si el total supera 20 minutos, el protocolo tiene pasos innecesarios.
Identificar qué se puede eliminar sin perder criterio.

---

## Día 8 — Sesión de revisión con el cliente

**Duración:** 30 minutos
**Objetivo:** El cliente lee el esquema completo y lo aprueba o ajusta.

**Agenda:**

```
0–5 min    → Presentar el esquema completo

5–20 min   → El cliente recorre cada regla IF → THEN
  Pregunta guía: "Si esta situación ocurre el lunes, ¿podés aplicar esta regla sin consultarme?"
  Para cada regla: sí / no / necesita ajuste

20–25 min  → Ajustes sobre la marcha si hay algo que no cierra

25–30 min  → Confirmar el protocolo de 15 minutos
  "¿Podés ejecutar estos 4 pasos sin que yo esté presente?"
  Si la respuesta es no: identificar el punto de duda y resolverlo.
```

**Al cierre del Día 8:**
- Esquema aprobado por el cliente
- Protocolo de 15 minutos validado
- Fecha de inicio de Fase 3 confirmada

---

## Día 9 — Ajustes finales y preparación de Fase 3

**Actividad principal:** Incorporar los ajustes del Día 8.

**Verificar contra `resources/delivery-standards.md`:**
- Cada THEN es específico y ejecutable
- No hay ambigüedad en ninguna zona de umbral
- El esquema pasa el test: "¿el staff puede aplicarlo sin consultar al consultor?"

---

## Día 10 — Entrega del esquema IF → THEN

**Mensaje de entrega:**

```
[Nombre],

Te comparto el esquema de decisión final.

Tiene [X] reglas para la VCV del bloque actual: [NOMBRE DE LA VCV].
El protocolo de validación lleva [X] minutos por semana.

La Fase 3 arranca el [FECHA].
En esa sesión, vos y el staff ejecutan el protocolo por primera vez con mi acompañamiento.
Después de eso, el sistema es del staff.

¿Hay algo del esquema que quieras revisar antes del [FECHA]?

Lisandro
```

**Señal de que el esquema no está listo para entregar:**
Si el cliente va a necesitar una explicación verbal para entender cualquier regla,
el esquema todavía no está escrito con suficiente claridad. Reescribir esa regla.
