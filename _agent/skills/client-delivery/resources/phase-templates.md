# Plantillas de Entregables — Tres Fases

Estas son las plantillas formales para los tres documentos de entrega de la Auditoría VCV.
Cada plantilla tiene una versión técnica (para Coordinador / PF) y una versión ejecutiva
(para Director Deportivo / Dueño de Academia).

Completar los campos entre `[corchetes]`. No entregar con corchetes sin completar.

---

## Entregable Fase 1 — Documento de Brecha

### Versión técnica (Coordinador / PF)

```
AUDITORÍA VCV — DOCUMENTO DE BRECHA
[NOMBRE DEL CLIENTE / ATLETA / INSTITUCIÓN]
Fecha: [DD/MM/AAAA]

─────────────────────────────────────────

BLOQUE ACTUAL
  Método:           [Wendler / Bloques / Ondulatorio / otro]
  Semana del ciclo: [Semana X de Y]
  Objetivo:         [Peaking / Volumen / Transición / otro]
  Próxima comp.:    [Fecha y nombre del evento]

─────────────────────────────────────────

LO QUE SE REGISTRA
  [Lista de métricas que el cliente registra actualmente]
  Ejemplo:
  - sRPE post-sesión (3x por semana)
  - AMRAP de semana 3 (cada ciclo Wendler)
  - HRV matutino (sin umbral definido)

LO QUE SE DECIDE
  [Descripción de cómo se toman las decisiones de ajuste hoy]
  Ejemplo:
  - El ajuste de TM se basa en la sensación del atleta post-AMRAP
  - No hay umbral escrito para definir cuándo el TM es incorrecto

─────────────────────────────────────────

LA BRECHA
  [Descripción concreta de la brecha en una oración]
  Ejemplo:
  "El AMRAP es el dato correcto pero no hay una regla escrita
   que defina qué resultado requiere acción sobre el TM."

VARIABLE CRÍTICA IDENTIFICADA
  [Nombre de la VCV del bloque actual]
  Ejemplo: AMRAP de Semana 3 — Sentadilla

EVIDENCIA DE CONSECUENCIA
  [Instancia concreta donde la brecha tuvo consecuencia visible]
  Ejemplo:
  - Ciclo de marzo: AMRAP = 3 reps. TM no se revisó.
    Ciclo siguiente: atleta falla la semana pesada.

─────────────────────────────────────────

PRÓXIMO PASO
  Fase 2 comienza el [FECHA].
  Entregable: Esquema IF → THEN con umbrales operativos para [VCV IDENTIFICADA].
```

## Entregable Fase 2 — Esquema IF → THEN

### Versión técnica (Coordinador / PF)

```
AUDITORÍA VCV — ESQUEMA DE DECISIÓN IF → THEN
[NOMBRE DEL CLIENTE / ATLETA]
Fecha: [DD/MM/AAAA]

─────────────────────────────────────────

VARIABLE CRÍTICA VALIDADA (VCV)
  Nombre:       [Nombre de la VCV]
  Métrica:      [Cómo se mide — número, test, observación]
  Frecuencia:   [Con qué frecuencia se registra]
  Responsable:  [Quién registra]

─────────────────────────────────────────

UMBRALES DE VARIACIÓN
  Zona OK (no intervenir):     [Rango o condición]
  Zona de atención (monitorear): [Rango o condición]
  Zona de intervención (actuar): [Rango o condición]

  Ejemplo para AMRAP Wendler:
  Zona OK:           AMRAP ≥ 5 reps con técnica limpia
  Zona de atención:  AMRAP = 3–4 reps
  Zona intervención: AMRAP ≤ 2 reps O fallo técnico

─────────────────────────────────────────

REGLAS DE DECISIÓN

  IF [condición de zona de atención]
  THEN [acción específica]
  Ejemplo:
  IF AMRAP = 3–4 reps
  THEN mantener TM actual en el próximo ciclo (no subir)
       + registrar RPE post-AMRAP para el siguiente ciclo

  IF [condición de zona de intervención]
  THEN [acción específica]
  Ejemplo:
  IF AMRAP ≤ 2 reps
  THEN reducir TM un 5% en el próximo ciclo
       + revisar carga de las 2 semanas previas al AMRAP

  IF [condición de zona OK]
  THEN [acción específica]
  Ejemplo:
  IF AMRAP ≥ 5 reps
  THEN subir TM según protocolo estándar (+2.5kg tren inferior / +1.25kg tren superior)

─────────────────────────────────────────

PROTOCOLO DE VALIDACIÓN SEMANAL
  Duración estimada: 15 minutos
  Responsable: [Nombre o rol]

  Paso 1: Registrar [VCV] al finalizar [sesión específica]
  Paso 2: Comparar con umbral
  Paso 3: Aplicar la regla IF → THEN correspondiente
  Paso 4: Documentar la decisión en [planilla / app / formato acordado]

─────────────────────────────────────────

NOTAS DE CONTEXTO
  [Cualquier consideración específica del atleta o el bloque que afecta la aplicación]
  Ejemplo:
  - Este umbral aplica para Sentadilla y Peso Muerto.
    Para Press de Banca, el AMRAP mínimo de zona OK es 4 reps (no 5).
  - Si hay acumulación de viajes en la semana del AMRAP, considerar fatiga externa
    como variable de contexto antes de intervenir.
```

  → [Acción del staff]

  Si el resultado está en zona de atención:
  → [Acción del staff]

  Si el resultado está fuera del rango:
  → [Acción del staff]

─────────────────────────────────────────

ROLES Y RESPONSABILIDADES
  Quién registra:    [Nombre / rol]
  Quién interpreta:  [Nombre / rol]

  - Una decisión documentada cada semana
  - Criterio consistente entre todos los preparadores
  - Respaldo de datos para cualquier conversación externa

## Entregable Fase 3 — Documento de Transferencia

```
AUDITORÍA VCV — DOCUMENTO DE TRANSFERENCIA
[NOMBRE DEL CLIENTE / INSTITUCIÓN]
Fecha: [DD/MM/AAAA]

─────────────────────────────────────────

RESUMEN DE LO INSTALADO
  VCV del bloque: [Nombre]
  Umbral de intervención: [Condición específica]
  Reglas IF → THEN: [Número de reglas escritas]
  Protocolo semanal: [Tiempo estimado]

─────────────────────────────────────────

PRIMERA VALIDACIÓN — RESULTADO
  Fecha de la primera validación autónoma: [DD/MM/AAAA]
  Dato registrado: [Valor de la VCV]
  Zona: [OK / Atención / Intervención]
  Decisión tomada: [Descripción de la acción]
  ¿El staff aplicó la regla de forma autónoma? [Sí / Con consulta / No]

─────────────────────────────────────────

AJUSTES POST-PRIMERA VALIDACIÓN
  [Cambios realizados al esquema IF → THEN luego de la primera aplicación]
  Si no hubo ajustes: "El esquema se aplicó sin modificaciones."

─────────────────────────────────────────

CONDICIONES DE AUTONOMÍA
  El staff está listo para operar de forma autónoma cuando:
  ✅ [Condición 1 verificada]
  ✅ [Condición 2 verificada]
  ✅ [Condición 3 verificada]

─────────────────────────────────────────

PRÓXIMOS CICLOS — RECOMENDACIONES
  [Qué revisar en el próximo bloque si el sistema continúa sin seguimiento]
  Ejemplo:
  - Revisar si la VCV sigue siendo la correcta al cambiar de bloque de fuerza a bloque de pico
  - Actualizar el umbral si el TM cambia más de un 10% respecto al bloque actual

─────────────────────────────────────────

CONTACTO POST-AUDITORÍA
  Si en las próximas 4 semanas el esquema genera una situación
  que el staff no sabe cómo resolver:
  [Forma de contacto y tiempo de respuesta prometido]

  Add-on de seguimiento mensual disponible: USD 200/mes
  Incluye 2 revisiones de validación + actualización del esquema.
```
