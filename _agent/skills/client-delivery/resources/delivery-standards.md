# Estándares de Calidad — Entregables por Fase

Usar antes de enviar cualquier entregable al cliente.
Un entregable no está listo hasta que pasa todos los criterios de su fase.

---

## Fase 1 — Documento de Brecha

### Criterios universales

```
✅ La brecha está descrita en UNA oración clara y sin ambigüedad
✅ La Variable Crítica está nombrada (no "la métrica principal" — el nombre real)
✅ Hay al menos una instancia concreta de consecuencia visible
✅ El documento tiene máximo 1 página (versión técnica) o media página (versión ejecutiva)
✅ No contiene recomendaciones de solución — solo diagnóstico
✅ No contiene jargon académico o de consultoría genérica
```

### Criterios por buyer

**Para Coordinador / PF:**
```
✅ El método de periodización está identificado correctamente
✅ La diferencia entre "lo que se registra" y "lo que se decide" es explícita
✅ El cliente puede leer el documento y decir "sí, este es exactamente mi problema"
```

**Para Director Deportivo:**
```
✅ La brecha está descrita como riesgo institucional, no como falla del preparador
✅ El lenguaje no asume conocimiento técnico de periodización
✅ La consecuencia está expresada en términos de impacto para el club (lesiones, credibilidad, resultados)
```

**Para Dueño de Academia:**
```
✅ La brecha está descrita como oportunidad perdida, no solo como problema
✅ Hay una referencia a cómo el problema afecta la percepción del servicio
✅ El documento no expone ni critica al staff individualmente
```

### Error más frecuente en Fase 1

> Identificar la brecha demasiado amplia.
> "No tienen un sistema de validación" no es una brecha — es una descripción genérica.
> La brecha correcta es: "El AMRAP de semana 3 no tiene un umbral que defina cuándo el TM debe revisarse."

## Fase 2 — Esquema IF → THEN

### Criterios universales

```
✅ La VCV tiene un nombre propio (no "la variable importante")
✅ El umbral de intervención tiene un número o condición específica — no un rango vago
✅ Cada regla IF tiene exactamente un THEN claro (no "evaluar" ni "considerar")
✅ El protocolo de 15 minutos tiene pasos enumerados, no descripciones generales
✅ El esquema puede aplicarse sin que Lisandro esté presente para interpretarlo
✅ Hay una nota de contexto si hay condiciones que afectan la aplicación de la regla
```

### Test de claridad del esquema

Antes de enviar, hacer este test mentalmente:

> "Si el preparador físico aplica esta regla sin haberme consultado,
> ¿va a tomar la decisión correcta el 90% de las veces?"

Si la respuesta es no → la regla todavía es ambigua. Reescribir.

### Criterios por buyer

**Para Coordinador / PF:**
```
✅ Los umbrales usan la misma métrica que el cliente ya registra
✅ Las reglas IF → THEN tienen los números exactos (reps, %, kg, valor de HRV)
✅ El protocolo de 15 minutos especifica qué herramienta / planilla usar
```

**Para Director Deportivo:**
```
✅ Los roles están asignados explícitamente (quién registra, quién interpreta, quién decide)
✅ El esquema incluye cómo se documenta cada decisión (para reporting futuro)
✅ El formato es el de la versión ejecutiva — no una tabla de umbrales técnicos
```

**Para Dueño de Academia:**
```
✅ El esquema incluye cómo se comunica el resultado al atleta o al padre
✅ El output del protocolo es algo que se puede mostrar externamente
✅ El formato es simple y visual — no denso en números
```

### Error más frecuente en Fase 2

> La regla THEN es vaga: "ajustar la carga según corresponda".
> Eso no es una regla — es la descripción del problema original.
> La regla correcta es: "reducir TM un 5% en el próximo ciclo."

## Fase 3 — Documento de Transferencia

### Criterios universales

```
✅ La primera validación ocurrió antes de redactar el documento
   (No se puede escribir el traspaso antes de que el staff haya ejecutado el esquema)
✅ El documento registra lo que pasó en la primera validación, no lo que se espera que pase
✅ Si hubo ajustes al esquema, están documentados con la razón
✅ Las condiciones de autonomía están verificadas — no asumidas
✅ Hay una recomendación concreta para el próximo ciclo
```

### Condiciones de autonomía — cómo verificarlas

Antes de redactar el documento de transferencia, verificar en persona o por mensaje:

```
Pregunta 1: "¿Podés decirme cuál es la VCV del bloque actual y por qué?"
  → Si puede articularlo con sus palabras: ✅
  → Si repite las palabras del documento sin entenderlo: ⚠️ Necesita otra sesión.

Pregunta 2: "¿Qué harías si el dato de esta semana cae en zona de atención?"
  → Si aplica la regla IF → THEN correctamente: ✅
  → Si dice "te consulto a vos": ❌ La transferencia no está completa.

Pregunta 3: "¿Dónde vas a registrar la decisión de esta semana?"
  → Si tiene un lugar definido: ✅
  → Si no lo sabe: ⚠️ El protocolo de 15 minutos necesita un paso más.
```

### Criterios por buyer

**Para Coordinador / PF:**
```
✅ El staff puede ejecutar el esquema sin acceso a Lisandro
✅ El documento de transferencia incluye las notas de contexto específicas del atleta
✅ La recomendación para el próximo ciclo está en lenguaje técnico
```

**Para Director Deportivo:**
```
✅ El documento incluye el protocolo de reunión técnica semanal instalado
✅ Los roles están verificados (se confirmó con cada persona del staff quién hace qué)
✅ La recomendación para el próximo ciclo incluye cómo mantener el criterio si hay cambio de staff
```

**Para Dueño de Academia:**
```
✅ Se hizo al menos una demo del sistema ante el dueño (no solo ante el PF)
✅ El documento incluye cómo se actualiza el sistema al cambiar de temporada
✅ Hay una descripción del output visible que la academia puede mostrar externamente
```

### Error más frecuente en Fase 3

> Redactar el documento de transferencia antes de la primera validación autónoma.
> La transferencia documenta lo que ocurrió, no lo que se planificó.
> Si el staff no ha ejecutado el esquema solo aunque sea una vez,
> la Fase 3 no terminó — solo se está planeando que termine.
