## Módulo: Señales de que necesitás la Auditoría VCV

### Para Coordinador de Rendimiento / Preparador Físico

```
Quizás la Auditoría VCV tiene sentido si:

→  Registrás datos pero las decisiones de ajuste siguen siendo intuitivas
→  Tu atleta llega al torneo fuera de pico sin una causa clara
→  No podés explicar con criterio por qué subiste (o no subiste) la carga esta semana
→  Tenés un cuerpo técnico y cada uno interpreta los mismos datos de forma distinta
→  Alguien — inversor, director, sponsor — te va a pedir que justifiques el proceso
```

### Para Director Deportivo / Manager del Club

```
Quizás la Auditoría VCV tiene sentido si:

→  Tus preparadores trabajan bien, pero sin un criterio común entre categorías
→  Tuviste lesiones que nadie anticipó a tiempo
→  No podés responder con datos concretos cuando la dirección pregunta por el proceso
→  Querés que el sistema funcione igual aunque cambie algún miembro del staff
→  Necesitás algo que demuestre la profesionalización del club hacia afuera
```

### Para Dueño de Academia

```
Quizás la Auditoría VCV tiene sentido si:

→  Tus atletas se entrenan bien pero no podés mostrar el sistema detrás
→  Los padres preguntan por qué y la respuesta es "confiá en nosotros"
→  Querés diferenciarte de otras academias con algo más que marketing
→  Necesitás reportes claros para atletas e inversores, no solo resultados
→  Querés justificar tu precio con un proceso verificable
```

## Módulo: Cómo funciona (3 fases) — Universal

```
La Auditoría VCV — 14 días

Fase 1 · Diagnóstico
Revisión del sistema actual. Identificación de la Variable Crítica del bloque.
Detección de la brecha entre lo que se registra y lo que se decide.

Fase 2 · Diseño
Umbrales de acción concretos. Reglas IF → THEN escritas.
Protocolo de validación semanal en menos de 15 minutos.

Fase 3 · Transferencia
El staff ejecuta el sistema de forma autónoma.
Sin dependencia del consultor.

Entregable: Un esquema de decisión aplicable desde el lunes siguiente.
```

## Notas de uso

- **Jerarquía visual:** H1 = una idea. Un párrafo = una idea. Nunca dos conceptos en el mismo módulo.
- **Bullets:** Máximo 5. Si hay más, dividir en dos secciones.
- **Longitud de CTAs:** Nunca más de 8 palabras en el botón. La descripción del CTA puede ir arriba.
- **Negrita:** Solo para la idea central de cada sección. Máximo 1 por bloque.
- **Estética:** IBM Plex Sans / Fira Code. Industrial. Sin gradientes. Sin ilustraciones decorativas.
- **Segmentación en la landing:** Si la landing tiene un menú o tabs por tipo de institución, usar los módulos de señales específicos por buyer. Si es una landing única, usar la Variante B del hero (universal) y los módulos de señales del Coordinador como default, con una sección específica para instituciones más abajo.
