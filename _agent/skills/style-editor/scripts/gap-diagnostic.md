# Script: Diagnóstico de Brecha Operativa

Usar cuando el usuario necesite analizar a un cliente potencial específico
o cuando llegue con descripción de un problema de staff técnico.

**Paso previo obligatorio: identificar el buyer antes de empezar el diagnóstico.**
El perfil del interlocutor determina qué preguntas hacer y cómo interpretar las respuestas.

---

## Identificación del buyer

Antes de las 4 preguntas de diagnóstico, determinar con quién se habla:

```
¿Quién es el interlocutor?
  → Coordinador de Rendimiento / Head of Performance
     → Diagnóstico técnico completo. Las 4 preguntas en orden.

  → Preparador Físico de equipo o categoría
     → Diagnóstico técnico completo. Foco en velocidad y criterio propio.
        Nota: puede no ser quien aprueba la compra. Ver "Nota estratégica" al final.

  → Director Deportivo / Manager del Club
     → Diagnóstico de gobernanza. Saltar a Pregunta 4 como apertura.
        El argumento es institucional, no técnico individual.

  → Dueño de Academia Deportiva
     → Diagnóstico de diferenciación. Ver sección específica al final.
```

---

## Proceso de diagnóstico para Coordinador / Preparador Físico (4 preguntas)

### Pregunta 1 — La regla de decisión

**Hacer:**
> "¿Qué dato te llevaría a cambiar la carga de tu atleta esta semana?"

**Evaluar la respuesta:**

```
¿La respuesta incluye un número, umbral o condición específica?
  → SÍ: Puede haber un sistema. Ir a Pregunta 2.
  → NO: Brecha Tipo A confirmada. La regla de decisión está ausente.
         Ir directamente a propuesta de Auditoría VCV.
```

---

### Pregunta 2 — El registro

**Hacer:**
> "¿Qué datos estás registrando actualmente, y con qué frecuencia?"

**Evaluar la respuesta:**

```
¿El registro es sistemático (aunque sea una planilla)?
  → SÍ: Hay material para trabajar. Ir a Pregunta 3.
  → NO: El problema es anterior a la brecha. El cliente necesita
         instalar el hábito de registro antes de una VCV.
         No es un cliente de Auditoría ahora mismo.
         → Derivar a simuladores IronLog como punto de entrada.

¿El cliente menciona más de 4–5 métricas distintas?
  → SÍ: Señal de Brecha Tipo B (exceso sin jerarquía). Ir a Pregunta 4.
  → NO: Probable Brecha Tipo A. Continuar.
```

---

### Pregunta 3 — La brecha en consecuencia

**Hacer:**
> "¿Hubo alguna instancia en el último ciclo donde el atleta llegó
> a una competencia sin estar donde esperabas?"

**Evaluar la respuesta:**

```
¿El cliente puede describir una instancia concreta?
  → SÍ: La brecha tiene consecuencia visible. El ROI es articulable.
         Registrar: qué pasó, cuándo, cuántas semanas afectó.
         Esta instancia es la base del argumento de propuesta.

  → NO (nunca pasó): Puede ser que el sistema funcione. Ser honesto:
         "Si nunca hubo un problema visible, quizás la prioridad es otra.
         ¿Hay alguna pregunta sobre el sistema de carga que no podés responder?"

  → NO (no recuerda): La brecha puede estar normalizada.
         "¿Podría haber pasado sin que te dieras cuenta?" → abre el espacio.
```

---

### Pregunta 4 — La gobernanza (solo si hay señal de Tipo B)

**Hacer:**
> "¿Todos los entrenadores del staff ajustan la carga con el mismo criterio?
> Si le pregunto a otro entrenador del equipo qué dato lo llevaría a cambiar
> la carga, ¿daría la misma respuesta que vos?"

**Evaluar la respuesta:**

```
¿La respuesta es "probablemente no" o "no lo sé"?
  → Brecha Tipo B confirmada. El problema es de gobernanza.
     Propuesta: Sistema de Gobernanza (Tier 2) o Auditoría como puerta de entrada.

¿La respuesta es "sí, tenemos criterio común"?
  → Verificar: "¿Ese criterio está escrito o es tácito?"
     Si es tácito: hay riesgo. Un cambio de staff lo pierde.
     Si está escrito: no hay brecha de gobernanza. Evaluar si hay brecha de VCV individual.
```

## Proceso de diagnóstico para Director Deportivo

El Director no va a responder preguntas técnicas. El diagnóstico es de gobernanza institucional.

### Pregunta de apertura

> "Si le preguntás a dos preparadores del staff qué dato los llevaría a cambiar
> la carga esta semana, ¿te darían la misma respuesta?"

```
"Probablemente no" / "No lo sé"
  → Brecha de gobernanza confirmada. El riesgo es sistémico.
  → Propuesta: Sistema de Gobernanza (Tier 2).

"Sí, todos trabajamos igual"
  → Verificar: "¿Ese criterio está escrito o es tácito?"
  → Si tácito: riesgo ante cambio de staff. Hay brecha latente.
```

### Seguimiento para cuantificar el costo institucional

> "¿Hubo lesiones en el último semestre que nadie anticipó?
> ¿Cómo respondió el club cuando alguien preguntó por qué pasó?"

```
Si describe una situación de conflicto o falta de explicación:
  → El ROI del sistema es credibilidad institucional y reducción de riesgo.
  → Argumento: "Un criterio institucional escrito convierte esa conversación
     en algo que podés responder con datos, no con intuición."
```

---

## Proceso de diagnóstico para Dueño de Academia

El Dueño piensa en competencia y valor percibido. El diagnóstico es de diferenciación.

### Pregunta de apertura

> "Cuando un padre o un atleta pregunta por qué el programa de entrenamiento
> está diseñado de la forma en que está, ¿podés mostrarle el proceso detrás?
> ¿O es una respuesta verbal que depende de que vos estés presente?"

```
"No tenemos eso documentado" / "Les explicamos"
  → Oportunidad de diferenciación. El proceso existe pero no es visible ni vendible.

"Tenemos reportes"
  → ¿Esos reportes incluyen criterio de decisión o solo datos?
  → Si solo datos: el proceso sigue siendo invisible.
```

### Seguimiento para cuantificar el valor de diferenciación

> "¿Hay academias en tu mercado que estén usando datos o dashboards como argumento de venta?
> ¿Cómo te estás diferenciando de ellas hoy?"

```
Si menciona competencia que ya usa datos:
  → Urgencia real. El sistema es defensa de posición, no solo mejora.

Si no hay competencia visible con datos:
  → Oportunidad de ser primero. Argumento de ventaja temprana.
```

## Output del diagnóstico

Al terminar, producir este resumen:

```
## Diagnóstico de Brecha — [NOMBRE CLIENTE / PERFIL]

**Buyer identificado:** [Coordinador / Director / PF / Dueño de Academia]
**Tipo de brecha identificada:** [Tipo A / Tipo B / Gobernanza / Diferenciación / Sin brecha / Pre-cliente]

**Evidencia:**
- [Respuesta clave que confirma la brecha]
- [Consecuencia concreta mencionada o inferida]

**Servicio recomendado:** [Auditoría VCV / Gobernanza / Ninguno por ahora]

**Argumento de ROI para este buyer:** [1–2 oraciones en el idioma de ese perfil]

**Próximo paso:** [Propuesta formal / Conversación de 20–30 min / Contenido de educación / Materiales para escalar internamente]
```

## Casos especiales

### El Preparador Físico no puede aprobar la compra

El PF encuentra el contenido pero el Director o Dueño firman el presupuesto.

Acción: completar el diagnóstico técnico con el PF, luego preguntarle:
> "¿Qué información necesitaría tu director / el dueño del centro para evaluar esto?
> Puedo prepararte algo para esa conversación."

Darle al PF materiales para vender la idea internamente. Su objeción no es el precio: es no tener los argumentos para subirlo a la cadena de decisión.

---

### Academia con Director Técnico y Preparador Físico distintos

Dos interlocutores, dos necesidades:
- El **PF** quiere criterio operativo → Auditoría VCV
- El **Director Técnico** quiere gobernanza y credibilidad → Sistema de Gobernanza

Si se habla con el Director primero: enmarcar en términos de consistencia del staff y riesgo institucional.
Si se habla con el PF primero: enmarcar en términos de autonomía y criterio propio. Luego equiparlo para la conversación con el Director.

---

### Cliente que ya hizo una Auditoría y el problema persiste

El problema evolucionó de Tipo A a Tipo B.
La regla existe pero el staff no la aplica de forma consistente.
Propuesta: Sistema de Gobernanza.

---

### Cliente con calendario de competencia en menos de 3 semanas

La Auditoría sigue siendo posible (14 días).
Pero el foco se limita a la VCV del bloque de tapering, no del sistema completo.
Mencionar esto en la propuesta: "alcance reducido al bloque actual."
